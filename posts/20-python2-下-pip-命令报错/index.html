20-python2 下 pip 命令报错

问题背景

  • 没有 pip 模块，那就下载一个 pip 包——当前最新版 setuptools-51.3.1

    • 在 setuptools-51.3.1 路径下打开 cmd ，使用 python setup.py install 命令手动安装模块，报错显示缺少 setuptools 模块，好家伙，要装 pip还需要先安装 setuptools。。

  • 下载好 setuptools 包——当前最新版 setuptools-51.3.1.tar.gz

    • 在 setuptools-51.3.1.tar.gz 解压目录下打开 cmd ，使用 python setup.py install 命令手动安装模块，又开始如下报错了，开始套娃。

最终解决方案

1、下载并运行 ez_setup.py 文件

下载ez_setup.py文件，并拷贝到 python2 的安装路径下，如下图

拷贝到安装路径下后，打开 powershell 或 cmd，运行 python ez_setup.py 这个命令即可开始编译构建，为接下来安装 pip 和 setuptools 模块做准备

打开 powershell 或 cmd，运行 python ez_setup.py 这个命令，结果如下

2、安装 setuptools 包

运行 ez_setup.py 后，python 安装路径下会出现一个 setuptools 压缩包——版本为 setuptools-33.1.1.zip（本机环境为 python 2.7）,将压缩包解压到当前路径，进入该文件夹，执行安装命令—— 使用 python setup.py install 命令手动安装模块，运行结果如下

运行结果和前一步骤的结果差不多，以防万一，还是执行一遍吧！

注意：前期无法安装上自己下载的高版本的 setuptools包可能是包版本和python版本不对应，所以安装报错。

3、安装 pip 包

PS C:\Python27\pip-20.3.3> python setup.py install
...... 此处省略 10000 行

4、添加环境变量

验证 pip 包

此时添加完环境变量后，将所有的 cmd 或 powershell 窗口关闭，输入 命令 pip list ，若列出已安装的包（pip 和 setuptools 的版本），则为安装成功，成功界面如下图

安装 第三方包 测试

上述图片中有一些 SSL 报错和连接失败报错，这是正常的，因为源包网站在国外，我们只需要改到国内就可以了。命令行运行 命令 pip config set global.index-url http://pypi.tuna.tsinghua.edu.cn/simple 即可一键修改好 pip 源 。

  • 安装 numpy ，运行 pip install numpy ，又开始报错

报错原因是没有添加信任主机，运行命令 pip config set global.trusted-host pypi.tuna.tsinghua.edu.cn 即可解决，也可以修改 pip 配置文件解决以上报错

  • 修改 pip 配置文件

修改内容为：

[global]
index-url = http://mirrors.aliyun.com/pypi/simple/
[install]
trusted-host = mirrors.aliyun.com

python2 安装 jupyter 报错

pip install ipykernel
# 会出现一些列报错，最新版的 IPython 不支持 Python 2.7，这时可先单独安装ipython，并指定旧的版本号进行安装，解决办法如下
pip install ipython==5.3.0
pip install tornado==4.5.3
pip install ipykernel
python -m ipykernel install --user
# 把python 2也加入到Jupyter的kernel里面
